# Build And Debug Policy

These rules apply to this project and should be migrated into equivalent instructions and skills when starting similar React + C++ desktop/web testbed projects.

## Debugging

VS Code debug configurations must be able to start immediately from a fresh checkout after dependency preparation.

Required behavior:

- Frontend debug checks dependencies before launching Vite.
- Frontend debug runs TypeScript typecheck before opening the browser.
- Backend debug bootstraps vcpkg when needed.
- Backend debug configures CMake before build.
- Backend debug builds the Debug executable before entering the debugger.
- Backend launch uses an explicit executable path or a generated launcher path that is verified by the prelaunch task.
- Prelaunch scripts must fail with a clear error when the executable is missing.

Avoid relying on editor-global selected launch targets when a stable path can be generated by the build system.

## Root Release Build

Every project of this style should expose a root build script:

```powershell
.\build.ps1
```

Required behavior:

- Check or install frontend dependencies.
- Build the production frontend bundle.
- Configure the backend build system.
- Build the Release backend executable.
- Fail clearly if the expected Release executable does not exist.

## Publishing

If external distribution is requested, provide a script that publishes artifacts under `/publish`.

Required behavior:

- Build Release artifacts unless explicitly skipped.
- Create a self-contained bundle folder.
- Include the backend executable and required runtime DLLs.
- Include the built frontend UI.
- Include start/install scripts.
- Create a versioned zip and a `latest` zip.
- Write `/docs` user and publishing guides.
